# 📂 GitHub vs 로컬 코드 변경점 비교 분석 보고서

현재 로컬의 개발 상태와 GitHub([mfcExam](https://github.com/dookidooki/mfcExam))의 최신 커밋 상태를 비교한 결과입니다.

---

## 1. 📋 파일별 주요 변경 사항

### ① CDlgImage.h (데이터 구조 및 선언)
*   **로컬 상태**: 
    *   `CPoint m_ptData[100];` (좌표 저장 배열) 추가됨.
    *   `int m_nDataCount;` (데이터 개수 관리 변수) 추가됨.
    *   `void drawData(CDC* pDC);` (그리기 전용 함수) 선언됨.
*   **GitHub 상태**: 위 변수와 함수 선언이 없음 (기초 다이얼로그 구조만 존재).

### ② CDlgImage.cpp (시각화 로직 구현)
*   **로컬 상태**: 
    *   `OnPaint`에서 `drawData(&dc)`를 호출하여 이미지 위에 도형을 덧그림.
    *   `drawData` 구현부: 빨간색(`RGB(255, 0, 0)`), 두께 `5`의 펜을 생성하여 `Ellipse`로 원을 그림.
*   **GitHub 상태**: `OnPaint`에서 이미지(`m_image.Draw`)만 출력하며, 별도의 도형 그리기 로직이 없음.

### ③ gPrjDlg.h (결과창 객체 관리)
*   **로컬 상태**: 결과 화면을 위한 두 번째 대화 상자 포인터 `CDlgImage* m_pDlgImgResult;`가 선언됨.
*   **GitHub 상태**: 메인 이미지용 `m_pDlgImage`만 존재함.

### ④ gPrjDlg.cpp (데이터 처리 및 화면 갱신)
*   **로컬 상태**:
    *   `OnInitDialog`: `m_pDlgImage`와 `m_pDlgImgResult` 두 개의 창을 나란히 생성(0,0 및 640,0).
    *   `OnBnClickedBtnTest`: 랜덤 좌표 생성 후, 검은 점(`0`)을 찾으면 **`m_pDlgImgResult->m_ptData`에 즉시 저장**하고 `Invalidate()`를 호출하여 화면을 갱신함.
*   **GitHub 상태**: 랜덤 좌표 생성 후 픽셀만 수정하고, 전체 스캔을 통해 콘솔에 좌표를 출력하는 기능만 들어있음 (화면 갱신 및 데이터 보관 로직 미흡).

## 2. 🚀 요약 및 결론

| 구분 | GitHub (기존) | 로컬 (현재) |
| :--- | :--- | :--- |
| **창 개수** | 1개 | 2개 (원본 / 결과) |
| **통신 방식** | 콘솔 출력 위주 | 배열 데이터 저장 및 전달 |
| **시각화** | 픽셀(점) 노출 | 빨간색 원(Ellipse) 오버레이 |
| **사용자 경험** | 정적 (결과 확인 불편) | 동적 (실시간 화면 반영) |

**결론**: 현재 로컬 코드는 GitHub의 단순 픽셀 조작 단계를 넘어, **"데이터 저장 -> 가공 -> 시각화 서비스"**로 이어지는 완전한 로직을 갖추고 있습니다. 특히 노란색이었던 원을 식별이 쉬운 빨간색 대형 원으로 개선한 점이 훌륭합니다.
